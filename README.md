## 1. 系統簡介

**智能合約處理系統** 讓您透過 AI 對話直接查詢合約內容，支援掃描檔（AI OCR）和原生文字 PDF，不需要手動翻閱合約或逐條比對條款。

### 可以做什麼

- **上傳合約**：上傳 PDF 或 DOCX 格式的合約文件，系統自動提取文字
- **問合約內容**：「保險條款是什麼？」、「付款方式有哪些規定？」
- **智能搜尋**：輸入合約描述，自動找到最相關的已處理合約並問答
- **快取加速**：系統儲存問答記錄，相同問題可直接複用，不需重新呼叫 AI
- **快取管理**：查看、刪除 Embedding 或問答快取，釋放空間或強制更新

## 2. 開始使用

### 環境需求

系統啟動前，請確認以下元件已安裝並正確設定：

| 元件 | 說明 |
|------|------|
| Ollama | 本機 LLM 服務（需預先下載所需模型） |
| LibreOffice | DOCX 轉 PDF 所需（處理 Word 文件時使用） |
| OCRFlux-3B | 本地 OCR 模型（處理掃描型 PDF） |
| CUDA GPU | Embedding 計算與 OCR 推理使用（無 GPU 可退到 CPU，但速度慢）|

### 路徑設定

首次使用前，需在下列四個程式檔案中將路徑修改為您的本機路徑：

| 檔案 | 需修改的常數 |
|------|-------------|
| `app.py` | `OCR_MODEL_PATH`、`TEXT_RESULT_DIR` |
| `doc_processor.py` | `TEXT_RESULT_DIR`、`TEMP_SPLIT_DIR` |
| `cache_manager.py` | `TEXT_RESULT_DIR` |
| `ui_helpers.py` | `TEXT_RESULT_DIR`、`ERRORLOG_DIR`、`DAILY_LOG_DIR` |

### 啟動系統

```bash
conda activate contract #啟動已經架好的環境
python app.py #需要是實際的檔案位置
```

啟動後開啟瀏覽器，前往 `http://localhost:8080`（或區域網路 IP）即可使用。

## 4. 上傳合約文件

### 支援格式

| 格式 | 說明 |
|------|------|
| `.pdf` | 原生文字 PDF（直接提取）或掃描檔（AI OCR）|
| `.docx` | Word 文件，系統自動透過 LibreOffice 轉為 PDF 後處理 |

### 上傳步驟

1. 切換到 **「📁 檔案上傳與手動搜尋」** Tab
2. 在 **Step 1** 區域上傳檔案
3. 點擊 **「▶️ 下一步：轉換與預覽」**
4. 系統自動讀取頁數並顯示預覽圖

### 同名偵測

若 `TEXT_RESULT_DIR` 下已有同名資料夾，系統會顯示警告訊息，但不阻止繼續。請確認是否要覆蓋舊資料。

---

## 5. 切割文件範圍

上傳成功後，系統顯示 **Step 2**，讓您選擇要處理的頁碼範圍。

### 操作說明

1. 輸入起始頁與結束頁（例如：第 3 頁到第 45 頁）
2. 點擊 **「▶️ 下一步：OCR 提取」**
3. 系統將指定範圍切割為暫存 PDF，等待後續處理

> 只處理合約本文，可略過封面、目錄、附件等不需要的頁面，提升處理速度。

---

## 6. OCR 文字提取

**Step 3** 負責從切割後的 PDF 提取文字內容。

### 兩種提取方式

| 方式 | 適用情境 | 說明 |
|------|---------|------|
| PDFPlumber | 原生文字 PDF | 速度快，直接提取，不耗費 GPU |
| OCRFlux-3B | 掃描型 PDF | AI 視覺模型辨識，需要 GPU，速度較慢 |

系統會**自動判斷**使用哪種方式，無需手動選擇。

### 完成後

提取完成後，文字內容儲存為 `.txt` 檔案至 `TEXT_RESULT_DIR/<合約名稱>/` 資料夾，並自動進入 **Step 4 問答**。

---

## 7. 與 AI 問答

**Step 4** 是問答的主要介面，系統自動為合約建立向量索引後，即可開始問答。

### 基本操作

1. 在輸入框輸入問題
2. 按 **「送出」** 或鍵盤 **Enter**
3. AI 分析問題、搜尋相關條款並生成回答

### 快取候選

每次送出問題時，系統會先在歷史問答快取中搜尋相似問題，顯示最多 3 筆候選答案：

```
您：付款條件是什麼？

系統：找到以下相似的歷史問答，是否直接使用？
  ① 付款方式為何？（相似度 92%）
  ② 費用如何支付？（相似度 75%）

您：選擇①（使用快取，不重新呼叫 LLM）
```

### 停止回答

AI 回答中途可點擊 **「■ 停止」** 按鈕截斷，可繼續輸入新問題。

### 選擇模型

右上角可切換 AI 模型（需透過 Ollama 預先下載）：

| 模型範例 | 適用情境 |
|---------|---------|
| qwen3:4b | 一般查詢，回應較快 |
| qwen3:14b | 複雜分析，準確度較高 |
| qwen3:32b | 最高精度，回應較慢 |

### 儲存與回報

| 按鈕 | 說明 |
|------|------|
| 儲存此問答 | 將本次問答加入 QA 快取，供下次相似問題使用 |
| 回報錯誤答案 | 將問答寫入 ERRORLOG，標記為不正確，不列入快取 |

## 8. 智能檢索回答

切換到 **「🔍 智能檢索回答」** Tab，可以用描述文字自動找到最相關的合約文件並直接問答，無需手動選擇合約。

### 操作步驟

1. 在「合約描述」欄輸入文件特徵（例如：「2023 年軟體採購合約，甲方為台灣科技公司」）
2. 在「問題」欄輸入想問的問題
3. 點擊 **「🔍 搜尋」**
4. 系統顯示最相似的合約候選，確認後自動載入並回答

### 切換候選文件

搜尋結果最多顯示 **5 個候選合約**，可點擊快速切換，針對不同合約問同一個問題。

### 更新檢索範圍

新增合約後，點擊 **「更新索引」** 讓智能搜尋包含最新的合約資料夾。

## 9. Embedding 快取管理

切換到 **「🗃️ Embedding 快取管理」** Tab。

### 什麼是 Embedding 快取

向量索引（Embedding）在合約首次載入時計算，並儲存為 JSON 檔案（`docstore.json`、`vector_store.json` 等）。快取存在時，下次載入相同合約可跳過計算，節省數十秒。

### 操作說明

| 操作 | 說明 |
|------|------|
| 重新整理 | 重新掃描並顯示所有快取的合約與佔用空間 |
| 刪除單一快取 | 從下拉選單選擇合約，刪除該合約的向量索引快取 |
| 清除全部快取 | 刪除所有合約的 Embedding 快取（更換 Embedding 模型後建議執行）|

> 刪除快取不影響 OCR 文字檔（`.txt`）和問答快取（`qa_cache.json`）。

## 10. 問答快取管理

切換到 **「💬 問答快取管理」** Tab。

### 什麼是問答快取

每次使用者點擊「儲存此問答」後，問答記錄會寫入對應合約資料夾的 `qa_cache.json`。每筆記錄包含問題、答案、使用模型、語言設定與時間戳記。

### 操作說明

| 操作 | 說明 |
|------|------|
| 選擇資料夾 | 從下拉選單選擇合約，顯示該合約所有問答記錄 |
| 刪除單筆 | 選擇特定問答條目（依 UUID）後刪除 |
| 清除此合約 | 刪除所選合約的所有問答快取 |
| 清除全部 | 刪除所有合約的問答快取 |

## 11. 歷史紀錄

在 **「📁 檔案上傳與手動搜尋」** Tab 內，可從歷史紀錄下拉選單選擇已處理過的合約，直接跳到 **Step 4 問答**，無需重新上傳和 OCR。

### 操作步驟

1. 從「歷史紀錄」下拉選單選擇合約名稱
2. 預覽圖顯示後，點擊 **「載入並前往問答」**
3. 系統自動載入向量索引（有快取則秒速完成），跳入 Step 4

## 12. 常見問題

### OCR 結果文字有誤或亂碼

- 掃描品質太低（低於 150 DPI）會影響 OCR 準確度
- 可嘗試只截取文字清晰的頁碼範圍再上傳
- 橫向掃描的文件可能需要先旋正再處理

---

### AI 沒有回應或回應很慢

- 確認 Ollama 服務是否正在運行（執行 `ollama list` 驗證）
- 較大的模型（14b、32b）回應較慢，可先用 4b 測試
- 若同時有 OCR 在執行，GPU 資源可能被佔用，建議等 OCR 完成後再問答

---

### 載入歷史合約時重新計算 Embedding 很慢

快取可能已被刪除。前往 **「🗃️ Embedding 快取管理」** Tab 確認。若快取不存在，下次載入時系統會自動重建，之後就會恢復快速載入。

---

### 想換用更好的 Embedding 模型

1. 在 `app.py` 的 RAG 模型載入區塊，修改 `model_name` 為新模型
2. 前往 **「🗃️ Embedding 快取管理」** → 清除全部快取
3. 重啟系統後，各合約首次載入時會自動以新模型重新計算向量

---

### 智能搜尋找不到剛上傳的合約

點擊 **「🔍 智能檢索回答」** Tab 右上方的 **「更新索引」** 按鈕，讓系統重新掃描 `TEXT_RESULT_DIR` 並納入新增的合約資料夾。

---

### 如何取消正在進行的 AI 回答

點擊對話區的 **「■ 停止」** 按鈕，系統會在下一個輸出片段中止生成，已生成的部分保留顯示。
